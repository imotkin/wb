# Задача L1.15

Рассмотреть следующий код и ответить на вопросы: к каким негативным последствиям он может привести и как это исправить?

Приведите корректный пример реализации.

```go
var justString string

func someFunc() {
  v := createHugeString(1 << 10)
  justString = v[:100]
}

func main() {
  someFunc()
}
```

Вопрос: что происходит с переменной justString?

---

Переменная `justString` будет ссылаться на исходный массив байт строки в переменной `v`, то есть сборщик мусора не сможет освободить память для переменной `v` из-за `justString`, пока переменная `justString` не будет сама освобождена им.

В качестве исправления текущего кода возможно создать новую и независимую копию для строки justString, а переменная `v` будет очищена сборщиком.

Для этого есть несколько способов:

## Использовать [strings.Clone](https://pkg.go.dev/strings#Clone)

```go
var justString string

func someFunc() {
  v := createHugeString(1 << 10)
  justString = strings.Clone(v[:100])
}

func main() {
  someFunc()
}
```

## Использовать массив байт для новой строки

```go
var justString string

func someFunc() {
  v := createHugeString(1 << 10)
  justString = string([]byte(v[:100]))
}

func main() {
  someFunc()
}
```

## Использовать массив рун для новой строки

```go
var justString string

func someFunc() {
  v := createHugeString(1 << 10)
  justString = string([]rune(v)[:100])
}

func main() {
  someFunc()
}
```

---

Примечание: первые два способа будут использовать первые 100 байт строки, а последний первые 100 символов, поэтому если в строке есть символы не входящие в ASCII и занимающие больше 1 байта, то между полученными строками будет разница, то есть в первых двух вариантах тогда возможно получить меньше 100 символов, из-за того, что один или нескольков символов занимают больше 1 байта, как было указано ранее.
