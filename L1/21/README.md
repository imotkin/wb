# L1.21 – Адаптер

Реализовать паттерн проектирования «Адаптер» на любом примере.

Описание: паттерн Adapter позволяет сконвертировать интерфейс одного класса в интерфейс другого, который ожидает клиент.

Продемонстрируйте на простом примере в Go: у вас есть существующий интерфейс (или структура) и другой, несовместимый по интерфейсу потребитель — напишите адаптер, который реализует нужный интерфейс и делегирует вызовы к встроенному объекту.

Поясните применимость паттерна, его плюсы и минусы, а также приведите реальные примеры использования.

[В созданном примере](main.go) был добавлен тип-адаптер `Adapter` для типа `Array`, чтобы этот адаптер соответствовал для интерфейса `Container`, который ожидается в функции `Print`. Данный паттерн можно использовать для того, чтобы получить ожидаемый тип для потребителя, в данном примере удалось делегировать вызов функции `First` для встроенного объекта `Array` и его метода `Get`. Плюс данного паттерна в гибкости для передачи значений, ожидаемых потребителем, однако есть минус, что из-за этого приходится вводить новый тип, объём кода увеличивается, и фактически возникает ещё один уровень для абстракции интерфейсов. В качестве реального примера использования можно привести адаптер для функции `http.HandlerFunc`, с помощью которого можно работать с интерфейсом `http.Handler`, то есть использовать любые функции, имеющие сигнатуру `func(w http.ResponseWriter, r *http.Request)` в качестве HTTP-хэндлера.
