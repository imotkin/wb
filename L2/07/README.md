Что выведет программа?

Объяснить работу конвейера с использованием select.

```go
package main

import (
	"fmt"
	"math/rand"
	"time"
)

func asChan(vs ...int) <-chan int {
	c := make(chan int)
	go func() {
		for _, v := range vs {
			c <- v
			time.Sleep(time.Duration(rand.Intn(1000)) * time.Millisecond)
		}
		close(c)
	}()
	return c
}

func merge(a, b <-chan int) <-chan int {
	c := make(chan int)
	go func() {
		for {
			select {
			case v, ok := <-a:
				if ok {
					c <- v
				} else {
					a = nil
				}
			case v, ok := <-b:
				if ok {
					c <- v
				} else {
					b = nil
				}
			}
			if a == nil && b == nil {
				close(c)
				return
			}
		}
	}()
	return c
}

func main() {
	rand.Seed(time.Now().Unix())
	a := asChan(1, 3, 5, 7)
	b := asChan(2, 4, 6, 8)
	c := merge(a, b)
	for v := range c {
		fmt.Print(v)
	}
}

```
---

## Вывод

```
12354768
```

Вывод не является детерминированным, то есть он зависит от случайного порядка чтения из каналов `a` и `b` в `select`.

## Объяснение

В данном примере реализованы две функции `asChan`, которая выполняет запись всех переданных ей чисел в возвращаемый канал, и `merge`, которая занимается записью значений из двух каналов `a` и `b` в одном канале `c`. Для этого используется `for select`, то есть `select` одновременно ожидает получение новых значений из каналов `a` и `b`, если такое значение получено, то оно записывается в канал `c`, при этом для проверки закрытия каналов `a`, `b` используется переменная `ok`, которая имеет значение `false` только в том случае, если канал из которого происходит чтение, был закрыт. При этом `select` сработает лишь для одного значения, поэтому необходимо использовать `for` для повторного чтения из каналов с помощью `select`.  Если для любого из каналов в функции `merge` выполняется `ok`, то в переменной для этого канала записывается `nil`. После закрытия двух каналов необходимо закрыть и общий канал, возвращаемый из функции `merge`, а также завершить выполнение горутины с `select` при помощи `return`.
