# L2.3

Что выведет программа?

Объяснить внутреннее устройство интерфейсов и их отличие от пустых интерфейсов.

```go
package main

import (
    "fmt"
    "os"
)

func Foo() error {
    var err *os.PathError = nil
    return err
}

func main() {
    err := Foo()
    fmt.Println(err)
    fmt.Println(err == nil)
}
```
---

## Вывод
Данная программа выведет следующее:
```
<nil>
false
```

## Объяснение
В Go интерфейс включает в себя два элемента: тип (T) и значение (V). V представляет собой конкретное значение определённого типа T. При сравнении интерфейса с `nil` будет `true` лишь в том случае, если тип и значение равны `nil`, а в данном примере из функции возвращается значения с интерфейсом `error`, где `T = *os.PathError` и `V = nil`, поэтому хотя значение V и равно `nil`, но сравнение выдаёт `false` из-за наличия типа внутри этой переменной интерфейса. Пустой интерфейс в Go это `interface{}` или его алиас `any`, он не имеет каких-либо методов, поэтому подходит для любого типа. Для всех интерфейсов, кроме пустых, возможно вызывать их методы, но для пустых такой возможности нет, а для получения исходного типа придётся, как и для других интерфейсов, использовать `type assertion`, например, в таком виде `i, ok := x.(int)` или с помощью `switch i := x.(type)`.
